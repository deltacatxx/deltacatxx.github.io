在锐龙四千系集显下深度播放器无法播放视频的总结

其实早在去年十一月份，我就已经发现了深度播放器无法播放视频的问题。具体表现为播放视频画面黑屏，但是声音能正常播放。一开始我没有在意，以为这么明显的 BUG 官方会很快修，而且后面我误以为已经修复了（实际上并不是所有视频都会有这个问题）。

但是到了去年十二月份，我发现我下载的物语系列动画依然无法播放时，我去质询了下内测群里的其他人有没有这个问题。他们建议我用终端运行下播放器看下有无输出报错。果不其然，在播放视频时出现了以下报错：

mesa: for the -simplifycfg-sink-common option: may only occur zero or one times!
mesa: for the -global-isel-abort option: may only occur zero or one times!
1
2

虽然我这方面了解不多，但是我知道 Mesa 是 Linux 下基于 AMD 开源显卡驱动开发的图形库。而刚好我的笔记本的显卡是 R7-4800U 的自带核显，那我猜测可能是显卡驱动问题。

于是乎我在 Github 上提了个 Issues (opens new window)，经过几天激烈的讨论后（其实是官方工作人员一般早上给答复而我一般晚上才有空回复），终于找到了问题所在！

（没有配图，我还没有搞图床）

我们已经定位到问题原因，因为amd4000系列，集显下，不支持h264硬解码导致，我们在后期会修复此问题。

问题已经确认，接下来就是等官方推送更新就是了。不过应该要等一段时间……
